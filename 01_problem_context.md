# Problem Context

Finance users need to review deductions, claims, invoices, and supporting evidence. The required documents may arrive through shared mailboxes, manual uploads, or different SharePoint folders by contacting the Back Office team.

Without stable linking logic, users spend time searching files or asking BO for evidence and documents.

## Digitalization goal

1. Incoming files are stored in the different location.
2. Finance records contain a stable ID. But, with respect to customers
3. Supporting documents can be found through that ID.
4. Missing and duplicate files were found and dealt.
5. The process can be reused and modified (if required) for additional customers later.

This repository does not include company data, internal SharePoint URLs, screenshots, exported flows, or credentials.
