# Problem Context

Finance users need to review deductions, claims, invoices, and supporting evidence. The required documents may arrive through shared mailboxes, manual uploads, or different SharePoint folders.

Without stable linking logic, users spend time searching files or asking colleagues for evidence.

## Digitalization goal

1. Incoming files are stored in the correct location.
2. Finance records contain a stable ID.
3. Supporting documents can be found through that ID.
4. Missing and duplicate files are visible.
5. The process can be reused for more customers later.

This public repository does not include company data, internal SharePoint URLs, screenshots, exported flows, or credentials.
